# Database Systems Design & UML Modeling for Tourism Management

## 📌 Project Overview
This project focuses on the design and modeling of conceptual and logical database systems using UML notations for tourism management scenarios. The work involved identifying entities, relationships, multiplicities, attributes, primary and foreign keys, and transforming business requirements into structured ERD models. The project demonstrates database design principles, system analysis, relational modeling, and logical data organization using industry-standard UML methodologies.

## 🛠️ Methodologies & Documentation
* **Methodologies:** Conceptual Modeling, Logical Modeling, Relational Database Design
* **Diagrams:** Enhanced Entity-Relationship Diagram (EERD), Logical Entity-Relationship Diagram (ERD)
* **Documentation:** Entity Data Dictionary, Specialisation Data Dictionary

## 📂 Case Studies

### Part A: Vacoliday (Conceptual EERD)
**Scenario:** Vacoliday is a touristic company that offers high-end, highly-customizable holidays in Vacomedita. The business manages various bookable accommodations (fully-integrated apartments and stand-alone rooms), multiple meal options, and a wide range of exciting bookable experiences led by specialized staff.
**Deliverables:**
* Produced a high-quality Conceptual Enhanced Entity Relationship Diagram (EERD) to support the company's data needs.
* Documented the architecture using an Entity Data Dictionary and a Specialisation Data Dictionary.

### Part B: Tourmato (Logical ERD)
**Scenario:** Tourmato is a touristic company that offers exciting "off-the-beaten track" visiting tours around several cities across Europe. The system requires tracking tour sessions (with specific dates, times, and prices) and managing customer bookings.
**Deliverables:**
* Mapped the provided conceptual data model onto a high-quality Logical Entity Relationship Diagram (ERD).
* Logically represented key business data needs as a set of interrelated tables, interconnected according to the strict rules of the relational model to be implementable.
